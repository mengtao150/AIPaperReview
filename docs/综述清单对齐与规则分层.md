# 综述清单对齐与规则分层

## 规则真源

本文件以 `综述清单.pdf`（更新至 2025-06-11）为审核规则真源，以 `文章结构.docx` 为期刊结构补充。模板发布时必须保存文件、页码、编号与版本；规则有冲突时，以编辑发布的模板版本为准。

PDF 中红色 `*` 是重点项，不是普通提示。系统应标记为 `priority=critical`、在工作台置顶、在报告中单列“重点复核项”；但仍由审稿人决定最终结论。

## 必须遵循的审稿顺序

```mermaid
flowchart LR
  A[解析稿件] --> B[Review/OA 候选判定]
  B --> C[审稿人确认文章类型]
  C --> D[Review 通用预检]
  D --> E[类型专属清单]
  E --> F[外部/人工核验]
  F --> G[中英文正式报告]
```

不能直接假设稿件为 Narrative Review。系统先判断主要价值是“基于文献的经验总结归纳”还是“原创设计”，然后按 PDF 决策图给出候选类型：Overview of Reviews、Scoping Review、Systematic Review、Rapid Review、Mini Review、Clinical Practice Review、Narrative Review。候选类型、证据和置信度必须展示，由审稿人确认后才应用专属模板。

## Review 通用预检

| 组别 | 清单要求 | 可执行边界 |
| --- | --- | --- |
| 重复率 | <20%，并检查直接复制/抄袭 | 需相似度服务；无外部语料时为“未核验” |
| PASS：临床价值/创新性 | 主题不夸大、rationale 站得住、类似研究与创新点明确 | 语义证据；“无类似研究”必须有外部检索支持 |
| PASS：科学性 | 标题/摘要/主体主题一致；结果具体；结论逻辑；期刊 scope；深度（gap、不一致、正负影响、证据匮乏）；广度（重要文献/RCT） | 跨章节、主张—证据和外部检索；不能用无证据断言代替 |
| PASS：可阅读性 | 结构清晰、逻辑通顺、内容聚焦、图表有助理解 | 给出具体章节/图表证据 |
| 图表 | 数量/正文引用/冗余；数据与正文/摘要/引文一致；题、统计信息、独立 footnotes 完整；可读性/分辨率；版权/CC BY | 前三项可自动辅助；图像质量和版权默认人工/外部核验 |
| 参考文献 | 缺引、赘述（一般单论点不超过 10 篇）、自引/本刊过引、正文/文后表一致、近三年文献（20% 为可配置建议）、未同行评议/撤稿/`publish online`、主张—来源不一致 | 书目状态、重要文献覆盖及来源语义需外部或人工核验 |
| 其他 | 缩写一致、格式、政审、伦理、字数 | 政审/伦理仅建立人工合规任务；字数按类型：Narrative/Clinical Practice <6000，Mini <3000 |

## Narrative Review 专属规则（按 PDF 原顺序）

1. 题目：重点突出、涵盖全面、定位明确为 Narrative Review；更新类需标 `updated`，检索范围是建议展示项。
2. 摘要：分别审查 Background and Objective、Methods（数据库、时间框架、语言）、Key Content and Findings、Conclusions（未来研究/临床/政策影响）。
3. 关键词：3–5 个，且与主题密切相关。
4. 引言：详细 Background、Rationale and knowledge gap、具体 Objective（`aim to`/`objective is`）、遵循 Narrative Review 清单声明。
5. 方法：
   - 检索日期和检索时间范围是两条独立规则，均精确到年月日；“近十年”“21 世纪”“2022 年”“不限时间”等均不充分；
   - 数据库充分性看综述目的，不机械要求数量；例如国际问题却无中文数据库应提示；
   - 搜索词覆盖范围、补充表或 `AND/NOT/OR` 策略、**正文和表格均报告**均须独立核验；
   - 纳排标准含研究类型与语言；
   - 筛选过程要说明执行者、独立性、共识方法；只在作者贡献中出现不算满足。
6. 主要内容：概述结果，评价纳入文献局限性/质量，说明综述自身优势与局限，提出未来研究，并总结图表。
7. 结论：明确对临床实践、政策或未来研究的意义。

## 可执行规则契约

除 `id`、`dimension`、`ruleType`、`severity`、`required` 外，每条规则必须有：

```json
{
  "source": {"file": "综述清单.pdf", "page": 4, "label": "5.4"},
  "reviewOrder": 54,
  "priority": "critical",
  "verificationMode": "deterministic|semantic|cross_document|external|human",
  "preconditions": ["article_type == narrative_review"],
  "evidenceRequirements": ["methods_text", "supplementary_table"]
}
```

报告内部状态为“到位、部分到位、未到位、无法判断、无需适用”，再映射至清单的“报告到位/不到位”。`external`/`human` 规则在没有证据时只能是“无法判断”，不得自动给出“到位”。
